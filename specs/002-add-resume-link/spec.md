# 기능 명세서: 이력서 링크 추가

**기능 브랜치**: `002-add-resume-link`  
**생성일**: 2025-11-17  
**상태**: 초안  
**입력**: 사용자 설명: "하단에 깃허브 링크 외에 이력서 링크도 하나 필요해"

## 사용자 시나리오 및 테스트 _(필수)_

### 사용자 스토리 1 - Contact 섹션에 이력서 링크 추가 (우선순위: P1)

포트폴리오 사이트 방문자가 Contact 섹션에서 개발자의 GitHub 프로필뿐만 아니라 이력서도 빠르게 확인할 수 있어야 합니다. 이력서 링크는 GitHub 링크와 함께 명확하게 표시되어야 합니다.

**이 우선순위의 이유**: 채용 담당자나 잠재 고용주가 개발자의 전문적인 배경을 빠르게 확인할 수 있는 핵심 기능입니다. 포트폴리오 사이트의 주요 목적인 취업/프로젝트 기회 창출을 위해 필수적입니다.

**독립 테스트**: Contact 섹션을 열어 이력서 버튼이 표시되는지, 클릭 시 올바른 링크로 이동하는지 확인할 수 있습니다.

**인수 시나리오**:

1. **Given** 사용자가 포트폴리오 홈페이지에 있을 때, **When** Contact 섹션으로 스크롤하면, **Then** "GitHub 프로필 보기" 버튼과 "이력서 보기" 버튼이 나란히 표시됨
2. **Given** Contact 섹션에서 이력서 버튼이 표시될 때, **When** 사용자가 이력서 버튼을 클릭하면, **Then** 이력서 페이지가 새 탭에서 열림
3. **Given** 모바일 화면에서 Contact 섹션을 볼 때, **When** 버튼들이 표시되면, **Then** 두 버튼이 반응형으로 적절하게 배치됨 (세로 또는 가로 레이아웃)

---

### 사용자 스토리 2 - 이력서 링크 설정 관리 (우선순위: P2)

개발자가 이력서 URL을 중앙 설정 파일에서 쉽게 관리하고 업데이트할 수 있어야 합니다.

**이 우선순위의 이유**: 이력서 URL이 변경될 때마다 여러 파일을 수정하지 않고 한 곳에서 관리할 수 있어 유지보수성이 향상됩니다.

**독립 테스트**: 설정 파일에서 이력서 URL을 변경하고 Contact 섹션에 올바르게 반영되는지 확인할 수 있습니다.

**인수 시나리오**:

1. **Given** 개발자가 site config 파일을 열었을 때, **When** resumeUrl 속성을 추가/수정하면, **Then** Contact 섹션의 이력서 버튼이 새 URL을 사용함
2. **Given** resumeUrl이 설정되어 있을 때, **When** 해당 URL이 유효한지 확인하면, **Then** 이력서 페이지에 접근할 수 있음

---

### 엣지 케이스

- 이력서 URL이 설정되지 않았거나 비어있을 때 어떻게 되나요?
  - 이력서 버튼이 표시되지 않거나 비활성화 상태로 표시
- 이력서 링크가 유효하지 않은 URL일 때는?
  - 브라우저의 기본 에러 처리에 위임 (404 페이지 등)
- 매우 긴 이력서 URL일 때 버튼 레이아웃은?
  - 버튼 텍스트는 고정되어 있으므로 URL 길이는 영향 없음

## 요구사항 _(필수)_

### 기능 요구사항

- **FR-001**: Contact 섹션은 반드시 "이력서 보기" 버튼을 표시해야 함
- **FR-002**: 이력서 버튼은 반드시 GitHub 버튼과 시각적으로 일관된 스타일을 가져야 함
- **FR-003**: 이력서 버튼은 반드시 클릭 시 새 탭에서 이력서 페이지를 열어야 함 (target="_blank", rel="noopener noreferrer")
- **FR-004**: 시스템은 반드시 site config 파일에 resumeUrl 속성을 추가하여 이력서 URL을 저장해야 함
- **FR-005**: 이력서 버튼은 반드시 모바일, 태블릿, 데스크탑에서 반응형으로 동작해야 함
- **FR-006**: 버튼 레이아웃은 반드시 화면 크기에 따라 적절하게 조정되어야 함 (모바일: 세로 스택, 데스크탑: 가로 배치)

### 주요 엔티티

- **사이트 설정 (Site Config)**: 포트폴리오 사이트의 전역 설정을 담고 있으며, 이력서 URL(resumeUrl)을 포함
  - 기존 속성: name, title, description, url, githubUrl, keywords
  - 신규 속성: resumeUrl (이력서 페이지 URL)

## 성공 기준 _(필수)_

### 측정 가능한 결과

- **SC-001**: 사용자는 Contact 섹션에서 5초 이내에 이력서 버튼을 찾아 클릭할 수 있음
- **SC-002**: 이력서 버튼은 모든 주요 브라우저(Chrome, Firefox, Safari, Edge)에서 정상 작동함
- **SC-003**: 이력서 버튼은 모바일(< 768px), 태블릿(768-1024px), 데스크탑(> 1024px) 모든 화면 크기에서 올바르게 표시됨
- **SC-004**: GitHub 버튼과 이력서 버튼의 스타일 일관성이 유지됨 (동일한 크기, 간격, 호버 효과)
- **SC-005**: 이력서 URL 변경 시 1개 파일(site config) 수정만으로 반영 가능

## 가정 사항

- 이력서는 외부 URL(예: Google Drive, Notion, PDF 호스팅 서비스)에 호스팅되어 있음
- 이력서는 PDF, 웹페이지, 또는 기타 브라우저에서 열 수 있는 형식임
- Contact 섹션은 이미 구현되어 있으며 GitHub 버튼이 표시되고 있음
- 버튼 컴포넌트(Button)는 이미 재사용 가능한 형태로 구현되어 있음

## 범위

### 포함

- Contact 섹션에 이력서 버튼 추가
- site config에 resumeUrl 추가
- 반응형 버튼 레이아웃
- 새 탭에서 이력서 열기

### 제외

- 이력서 파일 호스팅 (외부 서비스 사용 가정)
- 이력서 미리보기 기능
- 이력서 다운로드 기능 (외부 호스팅 서비스에서 제공)
- 다국어 이력서 지원
- 이력서 버전 관리
